# Customization

Practitioner SEO ships with opinionated defaults. This document explains
what you can customize and how.

---

## Methodology overrides

`SEO_METHODOLOGY.md` contains the evidence-based ranking model. Every claim
cites a source. If you disagree with a specific claim or your niche has
different dynamics, you can override specific sections in your
`SITE_CONFIG.md`:

```yaml
methodology_overrides:
  - section: "freshness"
    override: "Use 30-day fresh threshold"
    rationale: "Daily publication cadence; 90-day window is too loose"
  - section: "commercial_placement"
    override: "Contextual placement instead of bottom third"
    rationale: "Product reviews require inline purchase links for usability"
```

The skills read these overrides at runtime and adjust their behavior
accordingly. The override text is presented to Claude alongside the default
methodology, so Claude can apply the override while understanding the
context.

**What you can override:**
- Freshness thresholds
- Commercial section placement
- CWV threshold sensitivity
- CTR benchmark expectations
- Competitive ceiling assumptions

**What you should not override** (these are factual, not opinion):
- The evidence citations themselves (they're either accurate or not)
- Tool behavior (modify the tool code instead)
- Output format (the protocols control this)

## Freshness thresholds

The default framework:

| Age           | Status   |
|---------------|----------|
| < 90 days     | Fresh    |
| 90-180 days   | Monitor  |
| 180-365 days  | Stale    |
| > 365 days    | Critical |

To override, add to `SITE_CONFIG.md`:

```yaml
freshness_thresholds:
  fresh_days: 30
  monitor_days: 60
  stale_days: 180
```

**When to override:** News sites, rapidly changing niches, or sites where
Google shows a strong freshness preference for target queries.

**When to keep defaults:** Evergreen reference content, educational guides,
or niches where the top-ranking content is years old.

## Voice guide tuning

The voice guide generated by onboarding is a starting point. After using
the writer skill a few times, you'll notice patterns it gets right and
patterns it misses.

**Common adjustments:**
- Add specific sentence openers that characterize your writing
- Refine the "What NOT to do" list based on actual writer output
- Adjust the practitioner signal frequency (how often "in my experience"
  appears)
- Add page-type-specific guidance (e.g., "reviews should be more direct
  than guides")

The voice guide is a living document. Edit it whenever you notice the
writer producing something that does not sound like you.

## Forbidden words

The forbidden words list starts with common offenders seeded during
onboarding. Add to it whenever the writer uses a word or phrase you
dislike:

```markdown
## Site-specific additions

- "synergy" -- never appropriate for this site
- "leverage" (as a verb) -- corporate jargon
- "utilize" -- just say "use"
```

The writer skill scans every draft against this file and fixes violations
before delivering. The more specific your forbidden list, the less editing
you'll need to do.

## CMS quirk customization

CMS quirks are auto-populated during onboarding based on your CMS and
plugin combination. If you discover additional CMS-specific issues during
use, add them to `SITE_CONFIG.cms_quirks`:

```yaml
cms_quirks:
  schema_array_nesting: true
  inline_tag_concatenation: true
  # Custom quirk discovered during use:
  custom_note: "Theme injects duplicate Article schema on blog posts"
```

The skills will present custom quirk notes to Claude, who will account
for them in recommendations.

## Commercial model changes

If your commercial model changes (e.g., you add affiliate links to a
site that previously had none), update `SITE_CONFIG.content`:

```yaml
content:
  commercial_model: "affiliate"
  commercial_section_placement: "bottom_third"
```

And create/update the commercial list file referenced in
`SITE_CONFIG.context.commercial_list_file`.

The writer skill will start including commercial sections in new drafts.
Existing pages are not retroactively modified -- run the optimizer on
them to get commercial section recommendations.

---

## Adding a new pillar cluster

When you establish a new topic cluster:

1. **Update SITE_CONTEXT.md** with all new pages, their URLs, target
   keywords, cluster assignments, and link relationships
2. **Run a cluster diagnostic** on the new pillar: "Diagnose [pillar URL]
   for [seed keyword]"
3. **Update SITE_CONFIG if cannibalization risk changes** -- adding many
   new pages on adjacent queries may increase overlap risk

---

## Tool behavior customization

The MCP server tools themselves are not configurable via SITE_CONFIG.
If you need to modify tool behavior (e.g., change the user agent, adjust
timeout values, or modify scraping logic), edit the tool source files
directly in `mcp-server/src/practitioner_seo/tools/`.

After modifying tool code, reinstall the package:

```bash
cd mcp-server
pip install -e .
```

Then restart Claude Desktop.
